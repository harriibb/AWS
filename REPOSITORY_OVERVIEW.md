## 1. High-level purpose

This repository holds a **single-page marketing and academic deliverable** for an Oklahoma State University (OSU) **MIS Applied Security — DevOps Product Review** assignment. The page does two things at once:

1. **Campaign / advocacy:** Argues that **Micro Center** should open a store in **Stillwater, OK**, citing student population (~28K), tech majors (~3,500+), distance to the nearest Micro Center (Dallas, ~215 miles), and lack of local PC component retail.

2. **Product review:** Presents a structured written review of **Amazon Web Services** as a **CI/CD platform**, specifically **CodePipeline**, **CodeBuild**, and **CodeDeploy** — including strengths/weaknesses, numeric ratings, a conditional recommendation, and a “demonstration” section that describes a pipeline (GitHub → CodeBuild via `buildspec.yml` → deploy to **S3** static hosting) plus **SNS / CloudWatch** notifications.

The **same `index.html`** is both the reviewed “artifact” (static site) and the primary codebase. **`buildspec.yml`** is the minimal AWS CodeBuild specification that packages that HTML file for deployment.

There is **no** Node/React backend, **no** build toolchain in-repo (no `package.json`, no bundler), and **no** server-side code.

## 3. `buildspec.yml` — exact behavior

File is **AWS CodeBuild** buildspec **version 0.2**.

- **Phase:** `build` only (no `install` or `post_build`).
- **Commands:** Single `echo "Build started"` — no compile step because the “app” is static HTML.
- **Artifacts:**
  - `files`: `- index.html`
  - `discard-paths: yes` — artifact archive contains `index.html` at the **root** without preserving the original directory path.

**Implication for Claude:** A CodePipeline (as described inside `index.html`) can treat this repo’s output as a single file to push to S3 (or similar). The page text explicitly names **`buildspec.yml`** as the build definition for CodeBuild.

### 4.3 Boot sequence (`#boot`)

Full-screen overlay **before** main content:

1. **Phase 1:** Fake terminal types the command  
   `sudo ./deploy-campaign.sh --target=stillwater --mode=petition`  
   character by character into `#typeTarget`, with a blinking cursor, then shows “launching” output and animated loading dots.
2. **Phase 2:** Second terminal UI runs **`runScan()`**: reveals lines `l0`–`l6` sequentially, lights **boot segment** dots, animates a **progress bar** to 100%, then fades `#boot` out, adds `.revealed` to `#pageContent`, and **removes** the boot DOM node after transition.

Footer of boot UI references **“kernel v4.2.0 · MIS Applied Security”** (thematic, not a real kernel).

### 4.5 JavaScript API (what the page actually runs)

All global, no imports:

| Concern | Mechanism |
|--------|-----------|
| Slider | `goSlide(n)`, `setInterval` auto-rotate |
| Boot | `typeChar`, `showOutput`, `switchToPhase2`, `runScan`, timed `setTimeout` chain |
| Hero | `onScroll` — scales/fades `.hero-content`, parallax on `#heroBgParallax` and `#heroGlow` |
| Reveal on scroll | `IntersectionObserver` on `.scroll-reveal`, `.scroll-reveal-stagger`, `.zoom-reveal`; hero elements forced `.revealed` after 200ms |
| Split section | Scroll listener adjusts `transform` on `.slide-img > div` children |
| Mosaic | Scroll listener on `.mosaic-cell` (parallax offsets) |
| Photo reveal | `updatePhotoReveal` on scroll — uses **`blendProgress`** (smoothstep Hermite) for hero/grid blend; overlay opacity; per-cell scale/translate; per-panel translateY/opacity |

**Passive scroll listeners** are used where added (`{ passive: true }`).

### 5.1 Important URL strategy

Background images in CSS use **absolute URLs** to **GitHub raw** content, for example:

`https://raw.githubusercontent.com/harriibb/AWS/main/assets/<filename>.png`

**Implication:** Opening `index.html` from **file://** or from an S3 bucket **without** those images also hosted at the same relative paths will still load images **as long as** the browser can reach `raw.githubusercontent.com` and the branch/path remains valid. If the repo is renamed, made private, or paths change, **images break** unless CSS is updated to relative paths or a CDN.
