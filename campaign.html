Demonstration — Luke Hankins
Two Key Features

The following two features were demonstrated as part of this review to illustrate the practical capabilities of AWS as a CI/CD platform.

01

Automated Pipeline via AWS CodePipeline

We configured a three-stage pipeline (Source → Build → Deploy) connected to a GitHub repository. On every code push, CodePipeline automatically triggered CodeBuild to compile the app and run tests, then deployed the artifact to an S3-hosted static site.

Value: A developer commits code and within minutes it is live — no SSH, no manual uploads.

Source: GitHub Build: buildspec.yml Deploy: S3
02

Build Notifications via Amazon SNS + CloudWatch

We connected the pipeline to an SNS topic that sends email alerts on build success or failure. CloudWatch logs were configured to capture full CodeBuild output, allowing the team to diagnose errors without leaving the AWS console.

Value: Instant visibility into pipeline health — failed builds surface immediately via email.

Bring Micro Center Here

Stillwater is home to Oklahoma State University — one of the largest engineering and technology universities in the region. With over 3,500 students enrolled in Computer Science, Management Information Systems, and Electrical Engineering programs, the demand for quality tech retail is undeniable and currently completely unmet.

"The nearest Micro Center is in Dallas — a 3.5 hour drive. OSU students are forced to pay premium prices on Amazon or wait days for shipping on parts they need today. A Micro Center in Stillwater would serve students, faculty, and the broader north-central Oklahoma tech community from day one."
