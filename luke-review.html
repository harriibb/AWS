AWS Product
Review

Introduction, platform overview, and strengths & weaknesses of Amazon Web Services as a CI/CD tool.

Section A
Introduction
NamesBlake Harrison, Luke Hankins, Cole Sherer
PlatformAmazon Web Services — CodePipeline, CodeBuild, CodeDeploy
LaunchedAWS launched in 2006; CodePipeline introduced in 2015
Problem SolvedAutomates the build, test, and deploy lifecycle so development teams can ship software faster and more reliably without manual intervention
CompetitorsJenkins, TeamCity, GitHub Actions, GitLab CI/CD, Azure DevOps, CircleCI
Target UsersEnterprises and cloud-native teams already invested in the AWS ecosystem; scales from startups to Fortune 500
Key Features
⚙

CodePipeline

Visual, event-driven pipeline that orchestrates source → build → test → deploy stages automatically.

🔨

CodeBuild

Fully managed build service that compiles code, runs unit tests, and produces deployment-ready artifacts.

🚀

CodeDeploy

Automates deployments to EC2, Lambda, or on-prem servers with rollback support and traffic shifting.

🔗

Deep Integration

Native connections to S3, ECR, ECS, Lambda, CloudFormation, IAM, and CloudWatch out of the box.

🔔

SNS Notifications

Send instant email or SMS alerts on build success or failure through Amazon Simple Notification Service.

📊

CloudWatch Logs

Full build output captured in CloudWatch for real-time debugging and audit trails.

Use Case Example — Netflix

Netflix uses AWS extensively across its delivery infrastructure. Their engineering teams leverage CodePipeline and CodeDeploy alongside services like Lambda and ECS to push thousands of code changes per day globally with zero downtime. The fully managed nature of AWS CI/CD tools means Netflix engineers spend less time maintaining pipeline infrastructure and more time building product features — a critical advantage at their scale.

Section B
Strengths & Weaknesses
Strengths (+)
+ Free tier includes 1 active CodePipeline per month
+ No server maintenance — fully managed infrastructure
+ Native integration with 200+ AWS services
+ Scales automatically from small to enterprise workloads
+ Pay-per-use pricing — no upfront cost
+ Strong IAM-based security and compliance controls
+ GUI-based pipeline editor with drag-and-drop stages
+ Extensive documentation and community support
Areas for Improvement (–)
– Steep learning curve for AWS beginners
– Costs can escalate quickly without usage monitoring
– Heavily tied to AWS ecosystem — limited portability
– CodePipeline UI less intuitive than GitHub Actions YAML
– Debugging build failures requires navigating multiple consoles
– Limited built-in testing integrations vs competitors
– Free tier insufficient for multi-pipeline student projects
