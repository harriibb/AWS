Key Features
⚙️

CodePipeline

Visual, event-driven pipeline that orchestrates source → build → test → deploy stages automatically on every commit.

🔧

CodeBuild

Fully managed build service that compiles code, runs unit tests, and produces deployment-ready artifacts — no servers to manage.

🚀

CodeDeploy

Automates deployments to EC2, Lambda, or on-prem servers with built-in rollback support and traffic shifting.

🔗

Deep AWS Integration

Native connections to S3, ECR, ECS, Lambda, CloudFormation, IAM, and CloudWatch — all out of the box.

🔔

SNS Notifications

Instant email or SMS alerts on build success or failure via Amazon SNS, wired to EventBridge rules.

📊

CloudWatch Logs

Full build output captured in CloudWatch. Diagnose failures, trace errors, and monitor pipeline health in real time.

Use Case — Netflix

Netflix uses AWS extensively across its delivery infrastructure. Their engineering teams leverage CodePipeline and CodeDeploy alongside Lambda and ECS to push thousands of code changes per day globally. The fully managed nature of AWS CI/CD tools means engineers spend less time maintaining pipeline infrastructure and more time building product features — a critical advantage at their scale.

Section D
Recommendation
Conditionally Recommended

AWS CodePipeline is a powerful, production-grade CI/CD platform with industry-leading features and reliability. For student projects, it offers genuine real-world exposure — the same tooling used by Netflix, Airbnb, and NASA. However, it comes with a meaningful learning curve and costs that can catch newcomers off guard without careful monitoring.

We recommend AWS for student projects with conditions: teams should set up AWS billing alerts from day one, take advantage of AWS Educate / Academy programs for free credits, and pair the platform with clear documentation habits. The free tier covers one active pipeline — enough for a single-branch course project.

Compared to Jenkins (more control, more setup overhead) and TeamCity (better UI, higher licensing cost), AWS strikes a middle ground — it is the only option that requires zero infrastructure setup and provides a genuinely cloud-native experience. For MIS students headed toward cloud or enterprise IT roles, AWS is the most career-relevant choice of the three platforms.

Best For

Teams with AWS Educate access, cloud career focus, or projects deploying to EC2 / Lambda

Not Ideal For

Beginners with no cloud experience or projects where cost predictability is critical

Demonstration
Two Key Features

The following two features were demonstrated as part of this review to illustrate the practical capabilities of AWS as a CI/CD platform.

01

Automated Pipeline via AWS CodePipeline

We configured a three-stage pipeline (Source → Build → Deploy) connected to a GitHub repository. On every code push, CodePipeline automatically triggered CodeBuild to compile the app and run tests, then deployed the artifact to an S3-hosted static site.

Value: A developer commits code and within minutes it is live — no SSH, no manual uploads.

Source: GitHub Build: buildspec.yml Deploy: S3
02

Build Notifications via Amazon SNS + CloudWatch

We connected the pipeline to an SNS topic that sends email alerts on build success or failure. CloudWatch logs were configured to capture full CodeBuild output, allowing the team to diagnose errors without leaving the AWS console.

Value: Instant visibility into pipeline health — failed builds surface immediately via email.
